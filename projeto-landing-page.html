A transição do design para o código

Este projeto marca um ponto de inflexão na minha trajetória: a aplicação prática dos conhecimentos de front-end para construir algo do zero. A escolha de começar com HTML e CSS puros foi intencional — entender os fundamentos antes de adicionar camadas de abstração.

O projeto de uma landing page foi escolhido justamente por exigir atenção simultânea a hierarquia visual, responsividade, semântica HTML e organização de CSS — todos os pilares do desenvolvimento front-end.

Desktop
Mobile

O que foi construído

A página foi desenvolvida com foco em boas práticas desde o início: HTML semântico, CSS organizado em seções comentadas, variáveis customizadas e abordagem mobile-first para a responsividade.

  • HTML semântico com tags corretas para cada contexto
  • CSS com variáveis customizadas para paleta e tipografia
  • Layout responsivo com media queries mobile-first
  • Menu mobile funcional usando apenas o checkbox hack do CSS
  • Animações e transições suaves sem nenhum JavaScript
  • Scroll suave via scroll-behavior: smooth
:root { --clr-green: #10B981; --clr-blue: #2563EB; --font-display: 'Playfair Display'; --transition: 0.25s cubic-bezier(0.4,0,0.2,1); }

"Saber design me deu uma vantagem inesperada no front-end: já sei o que o código precisa produzir antes de começar a escrever."

Diferenciais do meu approach

Vir do design dá uma perspectiva diferente na hora de escrever código de interface. Enquanto aprendo as técnicas, já tenho intuição sobre espaçamento, hierarquia, contraste e ritmo visual — o que acelera decisões e evita erros comuns.
